Coin Shop Script för ESX
Jag vill att du skapar ett fullt fungerande FiveM-script till ESX som implementerar ett Coin-system och Coin-butik. Scriptet ska följa nedanstående specifikationer mycket noggrant:

🪙 Coin-system (ESX-baserat)
Coins ska inte ligga i inventory (för att undvika rån).

Varje spelare har ett separat coin-värde kopplat till sin identitet.

Coins ska endast kunna ges av admin i nuläget via kommandot:

bash
Copy
Edit
/givecoins [id] [antal]
Spelaren ska kunna se sitt coinsaldo via kommandot:

bash
Copy
Edit
/coinstatus
→ Detta ska visa en chat-notis med spelarens nuvarande coin-saldo.

🛠️ Adminmeny: /coinadmin
Endast tillgänglig för admins med rätt Steam Hex (via config).

När man skriver /coinadmin, öppnas en liknande meny som coinshop-menyn, men:

Visa först bara ett “+” i övre vänstra hörnet

När man klickar på plus:

Formulär öppnas med följande fält:

Item-namn (t.ex. weapon_pistol)

Visningsnamn (t.ex. "Pistol")

Bildlänk (för visning, t.ex. från fivemanage)

Coinpris

När man klickar “Lägg till vara”, sparas den till en config/datafil (helst JSON).

Tillbaka i adminvyn ska nu varan synas:

Med “Redigera”-knapp på varje produkt

Möjlighet att ta bort eller ändra produkten.

Alla produkter här speglas live till vanliga Coin-butiken.

🎯 Funktionella krav
All interaktion ska ske med NUI-menyer (HTML/CSS/JS).

UI ska vara inspirerat av bilden jag laddar upp (den har lila stil, men vår ska vara svart/grå med neonvit).

Scriptet ska vara:

Optimerat för ESX

Konfigurerbart (via config.lua eller JSON)

Modulärt (ha möjlighet att lägga till fler funktioner senare, t.ex. challenges för att tjäna coins)

All hantering av coins och produkter ska ske serverside.
